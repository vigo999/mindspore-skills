# Root-Cause Validation

After building a failure profile, validate the most likely cause groups against
real evidence.

Typical groups:

- environment or runtime mismatch
- library incompatibility
- config incompatibility
- dataset or input issue
- model asset issue
- checkpoint or resume issue
- backend or runtime issue
- operator-related issue

Every root-cause claim should carry:

- confidence
- supporting evidence
- a validation check
- a low-risk next action
