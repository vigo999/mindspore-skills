## 优化策略：质量优先，不是数量优先

### ❌ 不推荐：全量训练所有问题单

**原因**:
1. **数据质量参差不齐**: gitee 35K+ 问题单中，大量是重复、过时、非算子问题
2. **噪音大于信号**: 低质量数据会稀释高质量案例的权重，降低定界精准度
3. **维护成本高**: 全量数据需要持续清洗、去重、更新
4. **边际收益递减**: 前 100 个高质量案例已覆盖 80% 的常见问题模式

### ✅ 推荐：精准采样 + 持续迭代

**核心原则**: **每个分类 10-15 个高质量案例 > 1000 个低质量案例**

### 阶段 2: 增强误导性模式识别 (优先级: 高)

**问题**: 当前最大的定界失误来自"表面现象 ≠ 实际根因"

**已识别的误导模式**:
- allclose 失败 + 梯度为零 → 实际是 bprop 缺陷 (CS-001)
- AbstractProblem → 实际是编译器 pass 问题 (CS-009)
- 精度不一致 → 实际是基准框架版本差异 (CS-002)

**行动**:
1. **建立误导模式库** (`references/misleading_patterns.md`)
   - 结构: 表面错误 → 常见误判 → 正确定界路径 → 关键判断依据
   - 示例:
     ```markdown
     ### 误导模式 M-001: allclose 失败 + 梯度为零

     **表面现象**: 精度测试失败，梯度输出为零
     **常见误判**: 定界到 Kernel 精度问题
     **正确定界**: 反向传播 (bprop) 图结构问题
     **判断依据**:
     - 前向精度正常，仅反向异常 → bprop
     - 梯度为零而非小幅偏差 → 图结构而非精度
     - 导出反向 IR 检查 Select/DeadNode
     **参考案例**: CS-001 (#41932)
     ```

2. **在 SKILL.md 中强化二次验证**
   - Step 2 定界后，增加"误导模式检查"步骤
   - 如果匹配到误导模式，执行二次验证实验

**预期产出**: 识别 15-20 个误导模式，定界准确率从 70% 提升到 85%+

### 阶段 3: 算子特定知识深化 (优先级: 中)

**目标**: 为高频问题算子建立深度画像

**当前问题**: operator_profiles.md 只有路径和已知问题，缺少深度知识

**行动**:
1. **选择 Top 10 高频算子** (matmul, conv2d, gather, concat, add, reshape, transpose, cast, reduce, norm)

2. **为每个算子建立深度画像**:
   ```markdown
   ### matmul (深度画像)

   #### 源码结构
   - YAML: ops/op_def/yaml/matmul_op.yaml
   - Infer: ops/infer/ops_func_impl/matmul_ext.cc
   - Kernel (Ascend): ops/kernel/ascend/aclnn/.../batch_matmul_aclnn_kernel.cc
   - Bprop: ccsrc/frontend/expander/grad/grad_math_ops.cc

   #### 常见问题模式 (按频率排序)
   1. **CANN 升级后精度微调** (频率: 高)
      - 现象: 小幅偏差 (< 1e-3)
      - 根因: CANN matmul 内核优化
      - 修复: 重新基线
      - 案例: #41977

   2. **大 k 轴累加精度** (频率: 中)
      - 现象: fp16 下偶现精度超标
      - 根因: 累加误差
      - 修复: 提升中间精度或调整 tolerance
      - 案例: #41931

   3. **多卡场景 core dump** (频率: 低)
      - 现象: 偶现 segfault，栈中有 EnableDvmComm
      - 根因: 通信初始化多线程冲突
      - 修复: 加锁保护
      - 案例: #41961

   #### 快速诊断决策树
   ```
   matmul 问题
   ├─ 精度偏差 < 1e-3 → 检查 CANN 版本是否升级
   ├─ 精度偏差 > 1e-2 → 检查 k 轴大小 + dtype
   ├─ 偶现 core dump → 检查是否多卡 + 通信初始化
   └─ 性能退化 → 检查 shape 是否触发 fallback
   ```

   #### 定位技巧
   - 精度问题: 先用 `MS_DISABLE_KERNEL_BACKOFF=1` 隔离后端
   - 性能问题: 用 Profiler 检查是否调用了 ACLNN kernel
   - 多卡问题: 检查 HCCL 日志和通信组初始化时序
   ```

3. **从历史问题单中提取算子特定知识**
   - 搜索每个算子的所有历史问题
   - 归纳问题模式和频率
   - 提取快速诊断技巧

**预期产出**: Top 10 算子的深度画像，定位速度提升 50%

## 如何使用 Skill 达到最佳效果

### 使用前准备

1. **确保数据完整**
   ```bash
   # 检查必要的数据文件
   ls -lh /Users/claw/work/ms_debug/md_files/gitcode/issues/
   ls -lh /Users/claw/work/ms_debug/mindspore/mindspore/ops/

   # 检查 skill 文档
   ls -lh references/
   ```

2. **理解 skill 的能力边界**
   - ✅ 擅长: 常见问题模式的快速定界和定位
   - ✅ 擅长: 基于历史案例的修复建议
   - ⚠️ 有限: 全新的、从未见过的问题模式
   - ⚠️ 有限: 需要深度调试的复杂问题

### 标准使用流程

#### 1. 触发 Skill

当遇到 MindSpore 算子问题时，明确告诉 Claude:

```
我遇到一个 MindSpore 算子问题，请使用 mindspore-ops-debugger skill 帮我分析。

问题描述: [粘贴完整的错误信息和复现代码]
环境: 910B, CANN 20260105, MindSpore commit xxx
```

#### 2. 提供完整信息

Skill 需要以下信息才能准确定界:
- ✅ 完整的错误日志 (包括堆栈)
- ✅ 复现代码或最小化示例
- ✅ 环境信息 (设备、CANN 版本、MindSpore 版本)
- ✅ 预期行为 vs 实际行为
- ⚠️ 如果是精度问题，提供基准框架的结果

#### 3. 配合 Skill 的诊断流程

Skill 会按 6 步流程工作，你需要:

**Step 1 (问题分析)**:
- 如果 skill 询问缺失信息，及时补充

**Step 2 (定界)**:
- Skill 会搜索历史类似问题，可能需要你确认是否相似
- Skill 会给出定界结果，如果你有不同意见，明确指出

**Step 3 (定位)**:
- Skill 会导航到源码文件，你可以要求查看具体代码

**Step 4 (修复)**:
- Skill 会给出修复建议，你需要判断是否适用
- 如果不适用，告诉 skill 为什么，它会调整方案

**Step 5 (验证)**:
- 按 skill 的建议运行测试
- 将测试结果反馈给 skill

**Step 6 (测试补充)**:
- 如果需要补充测试，skill 会生成测试代码模板

#### 4. 处理定界失误

如果 skill 定界错误:

```
你的定界结果是"精度问题"，但我认为是"反向传播问题"，因为:
1. 前向精度正常
2. 梯度输出为零而非小幅偏差
3. 我导出了反向 IR，发现有 Select 节点

请重新定界。
```

Skill 会根据你的反馈调整定界，并记录这个误导模式。

#### 5. 利用历史案例

主动引导 skill 参考历史案例:

```
这个问题和 CS-001 (#41932) 很像，都是梯度为零。
请参考 CS-001 的定位和修复方法。
```

### 高级使用技巧

#### 技巧 1: 分阶段使用

不要一次性要求 skill 完成全部 6 步，而是分阶段:

```
# 第一轮: 只做定界
请先帮我定界这个问题，不要进行定位和修复。

# 第二轮: 定位
定界结果我确认了，现在请帮我定位到具体的源码文件。

# 第三轮: 修复
我已经看了源码，请给出修复建议。
```

这样可以在每个阶段及时纠正，避免错误累积。

#### 技巧 2: 对比验证

让 skill 给出多个可能的定界结果:

```
请给出 Top 3 最可能的定界结果，并说明每个的判断依据。
我会通过实验验证哪个是正确的。
```

#### 技巧 3: 利用算子画像

如果是高频算子:

```
这是 matmul 算子的问题。
请先查看 operator_profiles.md 中 matmul 的已知问题，
看是否匹配，再进行定界。
```

#### 技巧 4: 搜索历史问题

在定界前主动搜索:

```
在开始定界前，请先搜索 md_files/ 中是否有类似的问题单。
搜索关键词: "matmul", "精度", "910B"
```

#### 技巧 5: 反馈改进

每次使用后，如果发现问题:

```
这次定界失误了，原因是:
1. 错误信息中的"allclose"误导了定界
2. 实际根因是 bprop 中的 Select 操作

请将这个误导模式记录到 references/misleading_patterns.md
```

### 典型场景示例

#### 场景 1: 精度问题

```
问题: ops.pow 反向梯度为零
环境: 910B, CANN 20260105
错误: allclose_nparray 失败，data_me_error:[0.]

请使用 mindspore-ops-debugger skill 分析。
```

Skill 会:
1. 搜索历史问题 → 找到 CS-001 (#41932)
2. 识别误导模式 → "allclose 失败 + 梯度为零"
3. 正确定界 → 反向传播 (bprop)
4. 定位 → ccsrc/frontend/expander/bprop/grad_ops/grad_math_ops.cc
5. 修复建议 → 消除 Select 操作

#### 场景 2: 编译器问题

```

错误: ValueError: For 'X', condition.shape and input.shape need to broadcast

请使用 mindspore-ops-debugger skill 分析。
```

Skill 会:
1. 初步定界 → Shape 推导
2. 搜索历史 → 找到 CS-009 (#41973)
3. 识别误导模式 → "AbstractProblem 实际是编译器问题"
4. 重新定界 → 编译器/IR
5. 建议导出 IR → 检查 DeadNode
6. 修复建议 → 新增 switch_simplify pass

#### 场景 3: 全新问题

```
问题: 一个从未见过的新算子报错
环境: ...
错误: ...

请使用 mindspore-ops-debugger skill 分析。
```

Skill 会:
1. 搜索历史 → 未找到类似问题
2. 基于错误特征定界 → 给出最可能的分类
3. 提示: "这可能是新的问题模式，建议通过实验验证定界"
4. 给出验证实验建议
5. 根据实验结果调整定界

## 总结

### 核心原则

1. **质量 > 数量**: 100 个高质量案例 > 10000 个低质量案例
2. **深度 > 广度**: 深度理解 Top 10 算子 > 浅层覆盖 100 个算子
3. **迭代 > 一次性**: 持续小步迭代 > 一次性大规模训练
4. **反馈 > 预测**: 基于实际使用反馈优化 > 基于假设优化

### 优先级排序

1. **高优先级** (立即执行):
   - 补齐分类短板 (阶段 1)
   - 增强误导性模式识别 (阶段 2)
   - 建立验证与反馈循环 (阶段 5)

2. **中优先级** (1-2 周内):
   - 算子特定知识深化 (阶段 3)
   - 动态知识更新机制 (阶段 4)

3. **低优先级** (长期):
   - 全量 gitee 数据深度提取 (不推荐)

### 不要做的事

❌ 不要试图训练所有 35K+ gitee 问题单
❌ 不要追求 100% 的定界准确率 (边际成本太高)
❌ 不要忽视用户反馈 (实际使用是最好的测试)
❌ 不要过度自动化 (人工审核是质量保证)

### 预期效果

按照这个优化路线，预期:
- **1 周内**: 定界准确率提升到 80%+
- **1 月内**: 定界准确率提升到 85%+，定位速度提升 3x
- **3 月内**: 定界准确率提升到 90%+，50% 问题自动化解决
