# 算子代码检视清单

本文档提供算子代码检视的完整清单，按严重性分级。

## 1. 正确性 (Critical)

### 1.1 算法与数学

- [ ] **前向计算数学正确性**: 算法实现是否符合数学定义
- [ ] **梯度计算正确性**: 链式法则应用正确，特殊情况处理完善
- [ ] **数值稳定性**:
  - 溢出/下溢处理 (exp, log, pow)
  - NaN/Inf 检测和处理
  - 浮点累加精度损失
- [ ] **边界情况**:
  - 空张量 (shape 包含 0)
  - 零维张量 (scalar)
  - 单元素张量
  - 极大/极小值输入

### 1.2 Shape 推导

- [ ] **静态 shape 推导正确性**: 输出 shape 计算准确
- [ ] **动态 shape 支持**: 正确处理 -1 维度
- [ ] **Broadcasting 规则**: 符合 NumPy/PyTorch 标准
- [ ] **Reduction 维度**: keepdim 参数处理正确
- [ ] **特殊 shape**:
  - 标量输入 (shape = ())
  - 单维度 (shape = (n,))
  - 高维张量 (>4D)

### 1.3 Dtype 处理

- [ ] **支持的 dtype**: 已文档化并验证所有支持的类型
- [ ] **Dtype 提升规则**: 混合精度运算遵循标准规则
- [ ] **显式类型转换**: 避免隐式转换导致精度损失
- [ ] **整数溢出**: 整数运算结果不超出类型范围
- [ ] **特殊 dtype**:
  - Float16/BFloat16 精度损失
  - Complex 类型支持
  - Bool 类型逻辑运算

### 1.4 API 兼容性

- [ ] **签名匹配**: 参数名称、顺序、默认值与标准一致
- [ ] **参数验证**: 非法输入抛出清晰错误
- [ ] **向后兼容性**: API 变更不破坏现有代码
- [ ] **Deprecation 警告**: 废弃 API 有明确提示

## 2. 性能 (Major)

### 2.1 计算效率

- [ ] **算法复杂度**: 时间/空间复杂度合理
- [ ] **避免不必要拷贝**: 尽量使用 view 而非 copy
- [ ] **In-place 操作**: 安全时使用 in-place 减少内存
- [ ] **内存复用**: 临时 buffer 复用
- [ ] **Kernel fusion**: 识别 fusion 机会

### 2.2 NPU/CUDA 优化

- [ ] **Kernel launch 配置**: block/grid size 合理
- [ ] **内存访问模式**: 合并访问 (coalesced access)
- [ ] **Shared memory**: 合理使用 shared memory
- [ ] **Stream/Event 同步**: 异步执行优化
- [ ] **Bank conflict**: 避免 shared memory bank conflict

### 2.3 内存管理

- [ ] **内存分配模式**: 避免频繁小块分配
- [ ] **大张量处理**: 分块处理避免 OOM
- [ ] **临时 buffer**: 及时释放临时内存
- [ ] **内存泄漏**: 所有分配都有对应释放

## 3. 内存安全 (Critical for C++/CUDA)

### 3.1 指针安全

- [ ] **空指针检查**: 所有指针解引用前检查
- [ ] **数组边界检查**: 索引访问不越界
- [ ] **缓冲区溢出**: 写入操作不超出分配大小
- [ ] **Use-after-free**: 不访问已释放内存
- [ ] **智能指针**: C++11+ 优先使用智能指针

### 3.2 资源管理

- [ ] **RAII 模式**: 资源获取即初始化
- [ ] **异常安全**: throw 时无资源泄漏
- [ ] **错误路径清理**: 所有错误分支都清理资源
- [ ] **设备内存释放**: CUDA/NPU 内存正确释放

### 3.3 并发安全

- [ ] **线程安全**: 多线程访问无竞态条件
- [ ] **同步原语**: 正确使用 mutex/lock
- [ ] **原子操作**: 共享变量使用原子操作
- [ ] **死锁预防**: 锁顺序一致

## 4. 测试覆盖 (Major)

### 4.1 单元测试

- [ ] **基本功能**: 正常输入的正确性测试
- [ ] **边界情况**: 空张量、零维、单元素
- [ ] **错误输入**: 非法参数抛出正确异常
- [ ] **不同 dtype**: 所有支持的 dtype 都测试
- [ ] **不同 shape**: 1D, 2D, 3D, 高维
- [ ] **不同设备**: CPU, GPU, NPU (如适用)

### 4.2 梯度测试

- [ ] **前向-反向一致性**: 梯度计算与前向匹配
- [ ] **数值梯度对比**: gradcheck 通过
- [ ] **梯度累积**: 多次 backward 结果正确
- [ ] **高阶梯度**: 二阶梯度正确 (如需要)

### 4.3 性能测试

- [ ] **Benchmark 基准**: 性能符合预期
- [ ] **与参考实现对比**: 不慢于 PyTorch/NumPy
- [ ] **大规模输入**: 大张量性能测试

## 5. 文档 (Minor)

- [ ] **Docstring 完整**: 包含功能描述、参数、返回值
- [ ] **参数说明**: 每个参数的类型、含义、默认值
- [ ] **示例代码**: 提供可运行的使用示例
- [ ] **已知限制**: 说明不支持的情况
- [ ] **性能提示**: 说明性能相关注意事项

## 6. 代码质量 (Minor)

- [ ] **命名规范**: 变量/函数名清晰易懂
- [ ] **代码复用**: 避免重复代码
- [ ] **注释清晰**: 复杂逻辑有注释说明
- [ ] **错误处理**: 错误信息清晰有用
- [ ] **日志记录**: 关键路径有日志

## 检视流程建议

1. **第一遍**: 快速浏览，识别变更类型和影响范围
2. **第二遍**: 按 Critical 项逐项检查
3. **第三遍**: 按 Major 项检查
4. **第四遍**: 检查 Minor 项和代码质量
5. **总结**: 汇总问题，按严重性排序

## 严重性定义

- **Critical**: 必须修复，否则不能合并 (正确性、安全性)
- **Major**: 强烈建议修复 (性能、测试覆盖)
- **Minor**: 可选优化 (文档、代码质量)
- **Suggestion**: 改进建议 (最佳实践、可维护性)
